Phase 1 – Setup & Instalasi ✅

* Install Laravel ✅

  * `composer create-project laravel/laravel nama-project`
  * Konfigurasi .env (database, APP_NAME)

* Setup authentication (Laravel Breeze) ✅

  * Login, Register, Password Reset
  * Dashboard user

* Database setup ✅

  * Buat database di phpMyAdmin
  * Test koneksi database

* Migration dasar ✅

  * users (sudah ada)
  * password_resets

* Seeder user admin ✅

  * Buat user admin default

Phase 2 – Master Data (Kategori & Produk)

2.1 Kategori 

* Migration categories: id, name, slug, description, created_at, updated_at ✅
* Model Category: Relationship ke Product (hasMany) ✅
* CRUD Kategori

  * Index (tampil semua)
  * Create (form tambah)
  * Store (simpan)
  * Edit (form edit)
  * Update (update data)
  * Delete (hapus)
* Validation: Name required, unique; Slug auto generate

2.2 Produk

* Migration products: id, category_id, name, slug, description, price, stock, image, is_active, created_at, updated_at ✅
* Model Product: Relationship ke Category (belongsTo), Relationship ke OrderItem (hasMany) ✅
* CRUD Produk

  * Index dengan pagination
  * Create dengan upload gambar
  * Store dengan validasi
  * Show (detail produk)
  * Edit
  * Update (+ update gambar)
  * Delete (+ hapus gambar lama)
* Upload gambar: Storage link setup, Resize image (optional), Validasi file (max 2MB, jpg/png)

Phase 3 – Keranjang & Checkout

3.1 Session Cart

* Cart Controller: Add to cart (simpan di session), Update quantity, Remove item, Clear cart
* Cart logic: Cek stock sebelum add, Hitung subtotal per item, Hitung total keseluruhan

3.2 Checkout Process

* Migration orders: id, user_id, invoice_number, total, status, payment_method, shipping_address, notes, created_at, updated_at
* Migration order_items: id, order_id, product_id, quantity, price, subtotal
* Order Controller: Form checkout (input alamat, metode bayar), Generate invoice number otomatis, Simpan order + order items, Kurangi stock produk, Clear cart setelah order
* Model Order: Relationship ke User (belongsTo), Relationship ke OrderItem (hasMany)
* Model OrderItem: Relationship ke Order (belongsTo), Relationship ke Product (belongsTo)

3.3 Order History

* User order history: List semua order user, Detail order, Status tracking

3.4 Payment Setup

* Payment method list: Bank Transfer, E-Wallet, COD
* Payment status: pending, unpaid, paid, failed, expired

3.5 Payment Process

* Migration payments: id, order_id, payment_method, amount, status, paid_at, created_at, updated_at
* Payment Controller: Pilih metode pembayaran, Generate payment record, Update status pembayaran
* Payment logic: Validasi order, Cocokkan amount, Update status order setelah payment sukses

3.6 Payment Confirmation

* Manual payment confirmation (upload bukti transfer)
* Admin verifikasi pembayaran
* Update payment status: paid / failed
* Update order status otomatis setelah pembayaran valid

3.7 Payment History

* User payment history:
* List semua pembayaran
* Detail pembayaran
* Status pembayaran

Phase 4 – Admin Panel

4.1 Dashboard Admin

* Middleware IsAdmin (Cek role user di middleware, Protect admin routes)
* Dashboard statistics: Total produk, Total order hari ini, Total pendapatan bulan ini, Produk stock rendah
* Chart sederhana (optional): Order per hari (7 hari terakhir)

4.2 Kelola Produk (Admin)

* CRUD produk dari admin panel
* Filter by kategori
* Search produk
* Bulk action (aktif/nonaktif)

4.3 Kelola Order (Admin)

* List semua order: Filter by status, Filter by date, Search by invoice
* Detail order: Info customer, List items, Total pembayaran
* Update status order: Pending → Processing → Shipped → Completed, Atau: Cancelled

4.4 Kelola User (tambahan)

* List user
* Block/unblock user
* Ubah role (admin/customer)
